## SECTION 2: UI PROBLEMS — THE APP FEELS TOO "AI-GENERATED"

The current UI suffers from 7 specific aesthetic problems. Fix each one:

### UI PROBLEM 1 — GLASSMORPHISM EVERYWHERE
Every card uses the same semi-transparent blur-background glass effect.
This was trendy in 2021. It makes everything look samey and blurry.
Replace with: solid dark surfaces with precise 1px borders.
Specific values: background #0D0D0D for the page, #141414 for cards, 
#1C1C1C for elevated cards, #252525 for inputs and hover states.
Border: 1px solid rgba(255,255,255,0.06) for all cards.
No blur. No transparency. Flat, precise, terminal-like surfaces.

### UI PROBLEM 2 — TEAL/CYAN ACCENT COLOR IS OVERUSED
Every interactive element, badge, glow, and highlight is teal (#00D4AA or similar).
This is the default "AI fintech" color palette used by 50 other apps.
Replace the accent system with:
  - Primary accent: #E8FF47 (electric yellow-green) — used for active states, 
    selected nav items, and the most important number on each card
  - Confirmation/profit: #22C55E (clean green)
  - Loss/danger: #EF4444 (clean red)
  - Neutral data: #A3A3A3
  - LIVE badge: #FF3B30 (iOS-style red dot)
  - Everything else: white or gray
This palette feels like a Bloomberg terminal, not a crypto app.

### UI PROBLEM 3 — SIGNAL CONFIDENCE SHOWN AS PERCENTAGE, NOT SCORE
"88%" confidence as a percentage reads like a probability claim.
Change the display format to: score out of 100 shown as a horizontal 
bar with the number beside it. Example: [████████░░] 88
This feels more like a scoring system than a fake statistical claim.

### UI PROBLEM 4 — EVERY CARD HAS THE SAME HEIGHT AND LAYOUT
The dashboard bento grid shows 4 identical-sized metric cards.
This makes the layout feel like a template, not a designed product.
Make the grid asymmetric: 
  - Active Signals card: full-height left column, large number
  - Market Regime card: wide card spanning 2 columns, more detail visible
  - Portfolio Snapshot: right column, taller than the metrics
  - Signal Accuracy: medium width with the accuracy bar prominent
Use a CSS grid with named areas, not repeat(4, 1fr).

### UI PROBLEM 5 — NO VISUAL HIERARCHY BETWEEN INFORMATION TYPES
Every text element — headlines, numbers, labels, metadata, timestamps — 
is the same weight and similar size.
Establish a clear 4-level hierarchy:
  Level 1 (Hero numbers): 32px, weight 600, white
  Level 2 (Card titles): 15px, weight 500, white
  Level 3 (Labels and metadata): 12px, weight 400, #6B6B6B
  Level 4 (Timestamps and tags): 11px, weight 400, #4A4A4A
Apply this consistently everywhere. Numbers like "22,819.60" should be 
Level 1. Labels like "NIFTY 50" should be Level 3.

### UI PROBLEM 6 — SIGNAL CARDS IN RADAR LOOK LIKE BLOG POSTS
The Radar signal list renders each signal as a text-heavy list item.
It should look like a trading terminal signal feed.
For each signal card use this exact layout:
  Left edge: 3px color bar (green=bullish, red=bearish)
  First row: [COMPANY NAME large] [Signal type pill] [Score bar] [Date right-aligned]
  Second row: Headline text in a slightly larger, bolder font
  Third row: ₹ value in electric yellow | TAM or impact metric | right: sector tag
No excessive padding. Tight rows. Dense information. Terminal feel.

### UI PROBLEM 7 — NAVIGATION ACTIVE STATE USES BACKGROUND HIGHLIGHT
The active nav item uses a full-width background highlight (a selected row highlight).
This feels like a settings menu, not a product nav.
Replace with: 
  - Active item: text in electric yellow (#E8FF47), no background
  - A 2px left border in electric yellow on the active item
  - All inactive items: #6B6B6B text, no background
  - Hover: white text only, no background change
This is how Linear, Vercel dashboard, and Raycast handle nav.
